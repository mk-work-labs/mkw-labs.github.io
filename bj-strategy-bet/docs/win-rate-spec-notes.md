# 勝率表示の仕様検討ノート

## 背景

- 「勝率表示」は Phase 1 MVP の TODO（Specification.md §8）だが、仕様未確定のため**いったん保留中**
- 本ノートは決定のための判断材料を集約する目的。**決定はしていない**
- 決定後、改めて実装 TODO を起こしてから作業に入る

## 決めるべき論点

### 論点 1: 「勝率」の定義

- [ ] Prob(win) のみ（勝ちの確率だけを %表示）
- [ ] Prob(win) + 0.5 × Prob(push)（引き分けを半勝ち扱い）
- [ ] 期待値（EV, 配当込みのユニットあたり期待リターン）
- [ ] 複数を併記（例: 勝率 % と EV の 2 行）

### 論点 2: 計算方法（仕様書 §4.2.7）

- [ ] 方針A: 静的テーブル（事前計算した JSON を読み込む）
- [ ] 方針B: モンテカルロ（実行時にシミュレーション）
- [ ] 方針C: 解析計算（数式で厳密算出）

### 論点 3: 反映するルール範囲

- [ ] BJ 配当 1.5 倍
- [ ] ダブル成功時の倍率（ベット×2 で勝ち）
- [ ] スプリット後の期待値（複ハンド化）
- [ ] 推奨アクション前提 / 最適プレイ前提の違い

### 論点 4: 再計算タイミング

- [ ] 2 枚入力時のみ
- [ ] ヒット後の 3 枚目以降も再計算
- [ ] パフォーマンス予算（モンテカルロ選択時の反復回数上限など）

### 論点 5: EV 表示の扱い（仕様書 §4.2.2）

- [ ] Phase 1 では扱わない（勝率のみ）
- [ ] Phase 1 で勝率と並列に表示
- [ ] Phase 4 に後送り（仕様書の想定通り）

### 論点 6: カスタムストラテジー表（§4.2.8）との関係

- [ ] ユーザーのカスタム表を反映した勝率を出す（推奨アクションと整合）
- [ ] 常にデフォルト表での勝率を固定表示する（基準値として）

## 次のステップ

- 上記論点に対する回答を決めたら、実装 TODO を再開する
- 決定後は `bj-strategy-bet/src/logic/` 配下に勝率計算モジュールを追加予定
- UI 側は `StrategyPanel` の「勝率: —」プレースホルダ（`src/components/StrategyPanel/StrategyPanel.jsx`）を差し替える
